## 解题思路
**题目省流**：输入一个int数组nums，输出最长连续序列的长度。

**边界细节**：输入可能有重复的整数。

一开始觉得这题挺简单的。因为这题在哈希的分类里，我第一印象就是把这些整数作为键，值为一个bool变量，值为true，说明键在nums里存在。

但是我发现我对map的理解还是太浅了，在完成上述的想法之后，我脑海里有的想法，居然是最长连续序列在哈希表（数组）中是挨着的。

于是我第一个想法，是从0开始遍历，挨个检查键的值是否为ture，直到最大值，遍历的途中，保存最长连续序列的长度。但是我想不出来在复杂度要求下如何简单实现。

但实际上，挨着的键在哈希表中实际数组存储并不是挨着的，我那种想法是没完全理解哈希表的实际存储方式。

我之前想当然的把哈希当成一种特殊的数组，`mp[100]`,就好像真的要开100格数组一样，被数组留下的印象给骗了。

实际上哈希开的数组貌似只是n的1.5倍，所存的数组下标，是哈希函数算出来的。

我需要掌握**按数据遍历**的思维能力，而不是只会**值域遍历**。

然后是第二个想法：已知能按数据遍历的方法，从map里遍历出一个个键，要怎样找到最长连续序列呢？可以想到，**逻辑上键和键之间是挨着的**，所以可以判断这个键的前后都存在不存在，如果**前不存在，就说明这个键是起始点**。从起始点出发，一直找到后也不存在了，说明找到了一条连续序列。对每个起点都执行这样的操作，就可以得到最长连续序列的长度了。

## 最终代码
```go
func longestConsecutive(nums []int) int {
    //定义哈希
    mp := map[int] bool{}
    //循环加入哈希
    for _, v := range nums {
        mp[v] = true;
    }
    //从nums依次取出值，判断是否为起点，如果是起点则往后扫确认长度
    //错了，不能从nums里取出值，因为有重复值会增加复杂度，从哈希表里取出值就对了
    longestNum := 0
    for v := range mp {
        if !mp[v-1] {
            endV := v;//可以直接用v，但是为了之后增加需求（还有代码语义学），使用endV来保留起点
            currentNum := 1
            for mp[endV+1] {    //go使用for来替代while
                endV++
                currentNum++
            }
            if(longestNum < currentNum) {
                longestNum = currentNum
            }
        }
    }
    return longestNum
}
```
