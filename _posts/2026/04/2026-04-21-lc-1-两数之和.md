## 解题思路
写的第一道力扣题目，虽然题目很简单，但是还是花了不少时间。
慢慢熟悉刷题的感觉吧，我很久没刷过算法题了。写markdown也是，不太熟练。

**题目省流**：给了一个数组和目标数，找到数组内**两个数相加**的值等于目标数。输出这两个数的**数组下标**。

第一个想法是，用两层循环暴力枚举，用最熟的c++写了一个，提交之后过了。
但是我之前有了解过这道题，知道有用哈希表做的方法，但是我不知道哈希表怎么用，看了题解之后懂了。
之后又使用go语言重写了一次，发现比c++要简洁不少，但我还是写的不太熟练。
哈希表的思路是，我每遍历一个数，就把该数放进哈希表中，**数值为键，下标为值**。
下一个遍历的数x，寻找键为`target-x`的地方有没有存着数组下标就行了。
## 最终代码
```go
func twoSum(nums []int, target int) []int {
    hashTable :=map[int]int{}
    for i,x := range nums {
        if p, ok := hashTable[target-x]; ok {
            return []int{p, i}
        }
        hashTable[x] = i
    }
    return nil
}
```
## 本题收获
了解到了go语言的轻量和便捷

![两种语言](assets/img/Snipaste_2026-04-21_10-57-13.png)
*两种语言的用时和内存，可以看到go语言要比c语言强不少*

了解到一部分哈希表的知识：
1. c++有两个map，map底层是用**红黑树**，unordered_map底层是用**哈希表**（数组）。
2. 红黑树保持了**顺序**，还可以进行**范围查询**，查找和增删的复杂度是**稳定**的$O(\log n)$。
3. 哈希表的查找和增删的复杂度**平均** $O(1)$，最坏 $O(n)$ 。但内存占用比红黑树高，而且无序。
4. go语言内置的map是**使用最频繁**的哈希表，不提供红黑树map。

![如图所示](assets/img/Snipaste_2026-04-21_15-07-43.png)

了解到一部分迭代器的知识：
1. 在c++里，可以把迭代器想象成一个智能化的**指针**，他本质的作用，就和数组的**数值下标**一样
2. 迭代器在任何数据结构中，都可以通过`it++`来访问下一个元素，都可以通过`*it`来取值，这是数值下标无法做到的（只在数组有效）

了解到一部分go语法的知识：
1. `hashTable :=map[int]int{}`  声明一个key为int（中括号内），value为int的空map
2. `for i,x := range nums`  i为下标，x为值， `:= range`为语法，`nums`为被遍历的数据结构
3. 2里提到的i和x其实只是数组里返回这两个值，在不同的数据结构中，该语法返回的值都不一样
4. `if p, ok := hashTable[target-x]; ok` 这是go特色的带初始化语句的if判断，分号前是初始化，分号后是布尔判断部分
5. go的map支持**两个返回值**，p是通过键查到的值，ok是有没有查到。
6. 在go里map如果**没查到值，默认返回0**，但0这个键也可能有存着值，所以需要使用ok确认到底有没有查到。
