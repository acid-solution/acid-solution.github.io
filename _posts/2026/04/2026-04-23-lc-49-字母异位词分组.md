## 解题思路
**题目省流**：输入一个字符串数组，输出字母异位词分组后的一个**字符串数组**列表。
**字母异位词**：如"nat" 和 "tan"，使用相同的字母重新排列组合出来的词。
**边界细节**：输入为长度为0的字符串数组，空的也得分一组；输入为长度为1的字符串数组，单一个也分一组。
**思路历程**：由于题目是哈希分类里的，第一时间往哈希的方向去想，但是做题经验少了，只想到一些朦胧的东西。看了几眼题解的思路，大概懂了哈希分组题的特点。利用哈希分组的话，需要找到**同组的元素都共有的相同特征**，还要**反应在键上**。
本题中相同特征为，使用相同的字母。两种思路：**排序和计数**。
排序：相同字母排序后，得到的字符串一定相同。
计数：使用长度为26的字符串数组记录每个字母出现的次数,拼接到一块（要有分隔符，不然会有哈希冲突），得到的字符串也相同。
甚至还可以用质数乘积法，给为26个字母分别分配一个质数，计算出乘积作为键。每个大于1的自然数都可以唯一分解成质数的乘积。

通过这道题，我发现我的go语言基础还比较薄弱，需要针对性学习。
## 最终代码
方法一：排序
```go
func groupAnagrams(strs []string) [][]string {
    mp := map[string] []string{}
    for _, str := range strs {
        s := []byte(str)
        sort.Slice(s, func(i, j int) bool { return s[i] < s[j] })
        sortedStr := string(s)
        mp[sortedStr] = append(mp[sortedStr], str)
    }
    ans := make([][]string, 0, len(mp))
    for _, v := range mp {
        ans = append(ans, v)
    }
    return ans
}
```
方法二：计数
```go
func groupAnagrams(strs []string) [][]string {
    mp := map[[26]int] []string{}   //定义键为 每个字母出现次数 的 int数组
    for _,str := range strs {
        cnt := [26]int{}    //定义该int数组
        for _,b := range str {
            cnt[b-'a']++    //计算该int数组
        }
        mp[cnt] = append(mp[cnt], str)
    }
    ans := make([][]string,0,len(mp))   //初始化双层切片，用make提前申请内存
    for _,v := range mp {      //把存在map里的答案包装回双层切片
        ans = append(ans,v)
    }
    return ans
}
```
## 本题收获
了解到提示中的一些范围说明：
1. `strs.length <= 10^4`说明对字符串数组的操作时间复杂度最好在$O(N^2)$以下。
2. `strs[i].length <= 100`说明对字符串的操作时间复杂度可以在$O(N^3)$左右。
3. `strs.length >= 1`说明输入数组不会为空。
4. `strs[i].length >= 0`说明数组中可能包含空字符串。

了解到go语言的一些语法：
1. `mp := map[[26]int] []string{}`int数组也能当键？ 在 Go 中，数组是**值类型**且是**可比较**的。
2. 只要两个数组里的每一个元素都相等，这两个数组就被视为相等。
3. `[]string` 对标其他语言中的数组，但是这个叫**切片**，功能类似vector
4. `s := []byte(str)` string是**不可更改**的，而[]byte是**可以更改**的。
5. `sort.Slice` 可以对所有切片类型排序，和c++不同，后面必须提供比较函数，而且是参数是下标而不是值。
6. `func(i, j int) bool { return s[i] < s[j] }` 匿名函数，前面是参数，后面是返回值。
7. Go 强迫写`mp[k] = append(...)`是为了在不使用复杂指针的前提下，保证在内存扩容后依然能拿到正确的新地址。虽然啰嗦了一点，但降低了底层内存出错的概率
