### 每日内容总结
这周主要是从 Go 基础语法过渡到 Go 后端和并发里经常会遇到的概念。相比第一周，这周的内容更抽象，也更容易出现“看懂了，但不一定会写”的情况。

06/08 学了 interface 和组合思想。Go 的 interface 可以理解成一类能力的统一称呼，只要某个类型实现了接口要求的方法，就相当于实现了这个接口，不需要像 Java 那样显式写 implements。这里比较重要的是，接口一般不是越早抽象越好，而是需要解耦的时候再抽象。后端里常见的 Handler -> Service -> Repository 分层，也可以用小接口来组织依赖。组合思想这块也挺关键，Go 不走传统继承路线，而是更鼓励把能力拆小，然后按需要组合起来。

06/09 学了 pointer、值语义和引用语义。比较重要的收获是：Go 的函数传参永远是值传递，只是 slice、map、channel、function、interface、pointer 这些类型复制出来的值里带着对底层数据结构的引用，所以在函数里修改底层数据时，外面也能看到变化。struct 和 array 更偏纯值语义，如果要修改原对象或者避免大对象拷贝，就需要考虑传指针。这里也顺带理解了什么时候该用 pointer，比如需要修改原对象、表达可能不存在的值、避免大对象复制等。

06/10 学了 defer、panic、recover。defer 的核心作用是把资源申请和资源释放放在一起，减少忘记释放的概率。如果有多个 defer，会按照后进先出的顺序执行。panic 更像是程序进入了不正常状态，继续运行可能更危险；正常可预料的错误还是应该用 error。recover 必须放在 defer 里才能捕获 panic，而且只能捕获当前 goroutine 调用栈里的 panic。这里开始感觉到 defer 在资源释放和并发代码里会很常用。

06/11 学了 goroutine 基础。`go` 后面必须跟一个函数调用，启动之后这个函数会交给 Go runtime 调度执行。main 函数本身也是一个 goroutine，而且 main 结束时，其他子 goroutine 也会跟着结束，所以需要用 WaitGroup、channel、context 等方式等待或控制 goroutine。goroutine 的返回值也不能像普通函数那样直接拿，一般要通过 channel 传回来。这里还学到了 goroutine 泄漏的概念，也就是 goroutine 启动之后一直阻塞或无法退出，类似内存泄漏。

06/12 学了 channel 和 context。channel 可以让一个 goroutine 把结果安全地传给另一个 goroutine，也就是“不要通过共享内存来通信，而要通过通信来共享内存”。无缓冲 channel 在没人接收时发送会阻塞，有缓冲 channel 则是缓冲满了才阻塞。context 这块主要理解了它是用来控制一条任务链路生命周期的工具，可以通过 `ctx.Done()` 监听取消信号，通过 `ctx.Err()` 知道结束原因。HTTP 请求本身就带 context，后端里经常要把 context 一路传下去。

06/13 原计划是 Go 基础复盘加补题。实际记录里更多留下的是力扣复盘，Go 这边主要是把前几天的概念在脑子里串了一遍。

### 本周刷题收获
1. 快慢指针在链表中找中间节点这个问题上还是个比较好用的方法。
2. 53 和 560 这两题让我对“子数组区间和”问题有了更清楚的入口。以后遇到子数组区间和，可以先从 `prefix[j] - prefix[i]` 这个角度切入。关键不是马上套模板，而是固定右端点 `j`，思考左端点 `i` 应该如何处理。
3. 如果题目问的是固定目标，比如“和为 k 的子数组个数”，那就是在找满足 `prefix[i] = prefix[j] - k` 的历史前缀和，所以要用哈希表记录历史前缀和出现次数。如果题目问的是最优值，比如“最大子数组和”，那就是在所有可能的左端点里选一个最优的，可以理解成维护最小前缀和，也可以理解成 DP：维护以当前位置结尾的最大子数组和。
4. 合并链表的递归方法还挺有意思。
5. go的变量可以为函数，而且搞一个匿名函数放在内部再递归可以使用函数内部的变量，这样写起来最顺，比再写新函数和全局变量好。

## 本周Go串联理解
如果之后写一个用户接口，Handler 会接收 HTTP 请求，拿到请求自带的 context；Service 里处理业务逻辑；Repository 负责数据访问。Service 不一定要直接依赖某个具体 Repository，可以依赖一个小 interface。这样以后换实现或者做测试会方便一点。请求过程中如果要控制超时，就把 context 继续往下传；如果要启动并发任务，就用 goroutine；如果并发任务要返回结果，就用 channel；如果要释放资源，就用 defer。

这周学的东西不是孤立的：interface 负责解耦，pointer 负责对象修改和值语义理解，defer 负责收尾，goroutine 负责并发执行，channel 负责通信，context 负责生命周期控制。现在只是先把这些零件认识了一遍，真正能不能用好，要看后面写后端 demo 的时候能不能把它们自然地放进去。
